### Когда делать
Когда нужны:
- Авторизация (не все должны видеть данные)
- Загрузка XLS через интерфейс (без терминала)
- Форма ввода нецифровых показателей от руководителей
- Полный контроль над окружением

### Что нужно от пользователя перед стартом
- [ ] Выбрать и арендовать VPS (рекомендую: Hetzner CX22 ~4€/мес или Timeweb ~200₽/мес)
- [ ] Ubuntu 22.04 LTS
- [ ] Получить IP-адрес и root-доступ по SSH
- [ ] Опционально: домен (можно обойтись IP на первое время)

### Задачи на VPS
1. Установка Python 3.11, nginx, certbot (HTTPS)
2. Настройка systemd-сервиса для Streamlit
3. Перенос базы данных
4. Настройка nginx как reverse proxy (порт 8501 → 80/443)
5. Подключение домена + SSL-сертификат (Let's Encrypt)

**Важно:** на Linux/VPS xlrd работает нормально — win32com не нужен.
Скрипт импорта нужно адаптировать для Linux (заменить win32com на xlrd/openpyxl).

## 🗂️ АРХИТЕКТУРНЫЕ ДЕТАЛИ ДЛЯ ПАМЯТИ

### Почему win32com.client вместо xlrd
XLS-файлы из ProjectMate >2MB не читаются через xlrd (ложно-повреждённые).
Решение работает только на Windows. На VPS (Linux) использовать xlrd или openpyxl.

### decimal.Decimal в новом формате выгрузки (фев 2026+)
Числовые поля в новом шаблоне возвращаются как `decimal.Decimal`, а не `float`.
SQLite3 их не принимает. В `clean_value()` добавлен конвертор:
```python
import decimal
if isinstance(value, decimal.Decimal):
    return float(value)
```

### Поле «Подразделение» в XLS
Начиная с февраля 2026 поле заполнено в выгрузках (раньше было пустым).
В `import_data.py` оно маппится: `'Подразделение' → 'department'`.
Для старых периодов — брать из справочника staff через COALESCE:
```sql
COALESCE(t.department, s.work_group) AS work_group_resolved
```
**Вывод:** заполнять «Рабочая группа» в справочнике сотрудников больше не нужно —
она берётся из выгрузок. Справочник нужен только для окладов и уровней (этап ФОТ).

### Совместимость сумм (старый / новый формат)
- Старый формат: столбец «Сумма» → поле `amount`
- Новый формат: «Сумма, учтено» → `amount_recorded`; `amount` заполняется из `amount_recorded`
- Дашборд использует поле `amount` — работает с обоими форматами

### Курсы валют в данных (фев 2026+)
Поле `exchange_rate` теперь заполняется из столбца «Курс к учетной валюте (значения)».
Хардкод `USD_RATE, EUR_RATE = 88.0, 95.0` в дашборде пока остаётся,
но в будущем можно заменить на `SELECT AVG(exchange_rate) FROM time_entries WHERE currency=...`.

### Статус «Нома акта»
Опечатка в ProjectMate: столбец называется «Нома акта» (вместо «Номер акта»).
Маппится в поле `act_number`. Если в будущем исправят — добавить второй маппинг.

### Git LFS
Настроен для `*.sqlite`. Файл `.gitattributes` в корне dashboard/.
При клонировании репозитория на новой машине: `git lfs pull` для получения БД.
